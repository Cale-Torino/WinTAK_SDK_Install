Overview

WinTAK offers the ability to extend the application using buttons on the ribbon bar and new dockable windows. WinTAK utilizes the Managed Extensibility Framework (MEF) to add functionality. MEF follows a “convention over configuration” model. This document will walk you through adding a new dockable window and two buttons to WinTAK. This document assumes the use of Visual Studio 2015.

Getting Started

Creating the project

Open Visual Studio and select File -> New -> Project… In the New Project dialog, search for “wpf” in the search box in the upper right. Select the WPF User Control Library template and give it a name. For this document we will use SamplePlugin.

Create New Project

The project will come with one WPF UserControl class. This class will be the view of your dockable window.

Project Dependencies

The necessary WinTak Dependencies are available on the TAKMaps Artifactory NuGet server: https://artifactory.takmaps.com/artifactory/api/nuget/WinTAK

To setup Visual Studio to connect to PARs NuGet server perform the following steps:

  1. Install the nuget command line tool if you don't already have it (https://docs.microsoft.com/en-us/nuget/install-nuget-client-tools)
  2. Open the command prompt
  3. Run the following command to add the Artifactory Nuget package source where USERNAME and PASSWORD are your credentials for the PAR Artifactory website

nuget sources Add -Name Artifactory -Source https://artifactory.takmaps.com/artifactory/api/nuget/WinTAK -username <USERNAME> -password <PASSWORD>

Importing WinTak Dependencies

  1. Open Tools->NuGet Package Manager->Manage NuGet Packages for Solution.
  2. Select Browse and make sure Package Source is set to "Artifactory" or "All".
  3. Search for WinTak.
  4. Select WinTak-Dependencies package and install the version you wish to build your plugin against.
WinTak Dependencies installed via NuGet

In addition to the WinTAK Dependencies you will need to add one other reference. Right click the project in the Solution Explorer and select Add -> Reference… You’ll need to add a reference to System.ComponentModel.Composition. You can find System.ComponentModel.Composition in the Assemblies node.

Creating the Dockable Window

The dockable window will be the view that is displayed whenever your window is shown.

Creating the view

In your project there should be a file named UserControl1.xaml, rename this to SampleView.xaml and then double click SampleView.xaml to open the view. The view will be empty with only a Grid control within it. We will add one Button control and a TextBlock to the view.

Add the following XAML to the view inside of the Grid element.

<StackPanel>
<Button Content="Increase counter "
Command="{Binding IncreaseCounterCommand}" />
<TextBlock Text="{Binding Counter}" />
</StackPanel>

Creating the DockPane

The DockPane class provides the ability to Hide/Show your dockable window and serves as the ViewModel for your view. Right click the project in the Solution Explorer and select Add -> New Item. Select Class and name it SampleDockPane.

Adding the DockPane class

Add the following using statements to the class

using System.ComponentModel.Composition;
using System.Windows.Input;
using WinTak.Framework.Docking;
using WinTak.Framework.Docking.Attributes;

The SampleDockPane class should inherit from the DockPane class.

class SampleDockPane : DockPane
{
}

The SampleDockPane class will hold a Counter property that will increase every time the button in the view is clicked. To increase the counter another IncreaseCounterCommand property is used that the button binds to. Add the following code to your SampleDockPane class.

class SampleDockPane : DockPane
{
private int _counter;
public SampleDockPane()
{
var command = new ExecutedCommand();
command.Executed += OnCommandExecuted;
IncreaseCounterCommand = command;
}
public ICommand IncreaseCounterCommand { get; private set; }
public int Counter
{
get { return _counter; }
set { SetProperty(ref _counter, value); }
}
private void OnCommandExecuted(object sender, EventArgs e)
{
Counter++;
}
private class ExecutedCommand : ICommand
{
public event EventHandler CanExecuteChanged;
public event EventHandler Executed;
public bool CanExecute(object parameter)
{
return true;
}
public void Execute(object parameter)
{
var handler = Executed;
if (handler != null)
{
handler(this, EventArgs.Empty);
}
}
}
}

Attributes

Attributes are what MEF looks for when loading assemblies. When a class has the ExportAttribute assigned to it, MEF will add the class to the container of all classes it knows about.

Add the following attribute to the class:

[DockPane(ID, "Sample", Content = typeof(SampleView))]

And the following constant to the class:

internal const string ID = "Sample_SampleDockPane";

Create a Button to Open Your Dockable Window

Buttons are placed on the Ribbon panel of WinTAK. Buttons can be used to initiate map interaction, open dockable windows, or anything else that needs activated.

Add a new SampleButton class to the project.

Adding the Button class

Add the following using statements to the class

using System.ComponentModel.Composition;
using WinTak.Framework.Docking;
using WinTak.Framework.Tools;
using WinTak.Framework.Tools.Attributes;

The SampleButton class should inherit from the Button class.

class SampleButton : Button
{
}

Attributes

Like the DockPane, buttons are registered with MEF using class attributes. The following attributes will be used with the button.

[Button("Sample_SampleButton", "Sample Plugin",
LargeImage = "pack://application:,,,/Sample;component/Assets/Settings.svg",
SmallImage = "pack://application:,,,/Sample;component/Assets/Settings_24x24.png")]

The Button registers images that will be used for the Ribbon bar. These images are registered with the LargeImage and SmallImage metadata properties. This example uses two images found in the Assets folder of the project. The images should have a build action of Resource.

Opening the dockable window

When the button is clicked it will open the dockable window. The window can be opened with the IDockingManager. The button uses MEF to get the single instance of IDockingManager. Using the ImportingConstructor attribute on the constructor, MEF will inject in required dependencies.

private IDockingManager _dockingManager;
[ImportingConstructor]
public SampleButton(IDockingManager dockingManager)
{
_dockingManager = dockingManager;
}

When the button is clicked it will open the DockPane. Override the OnClicked method to open the DockPane.

protected override void OnClick()
{
base.OnClick();
var pane = _dockingManager.GetDockPane(SampleDockPane.ID);
if (pane != null)
pane.Activate();
}

Lazy Loading

WinTAK uses a lazy loading approach with MEF. Buttons and DockPanes will not get created until they are needed. This speeds up startup time of the application. Buttons will be created when they are clicked and DockPanes will be created the first time they are obtained from the IDockingManager.

Modules

If you have services in your plugin that need to be initialized at startup it's best to start those services from a Module.

Add a new SampleModule class to your project.

Adding the Module class

Add the following using statements to the class

using System.ComponentModel.Composition;
using Prism.Events;
using Prism.Mef.Modularity;
using Prism.Modularity;

The SampleModule class should implement the IModule interface and should be decorated with the ModuleExport attribute in order to be recognized by MEF.

[ModuleExport(typeof(SampleModule), InitializationMode = InitializationMode.WhenAvailable)]
class SampleModule : IModule
{
}

The IModule interface will require the class to implement an Initialize method. This method will be called on startup to notify the module that it has been initialized. This may be a good place to startup services that your plugin depends on or register for events that your module is interested in.

In this example we will register for an event that a file or folder has been drag and dropped onto WinTAK. Update the class with the following code:

[ModuleExport(typeof(SampleModule), InitializationMode = InitializationMode.WhenAvailable)]
class SampleModule : IModule
{
private readonly IEventAggregator _eventAggregator;
[ImportingConstructor]
public SampleModule(IEventAggregator eventAggregator)
{
_eventAggregator = eventAggregator;
}
// Modules will be initialized during startup. Any work that needs to be done at startup can
// be initiated from here.
public void Initialize()
{
_eventAggregator.GetEvent<FileDroppedEvent>().Subscribe(OnFileDropped);
}
private void OnFileDropped(FileDroppedArguments args)
{
// The user has drag and dropped a file onto the WinTak window
}
}

In your handler method for an event like FileDroppedEvent it is important to mark the event as handled if and only if the file is relevant to your plugin.

Adding the project to WinTAK

WinTAK will load assemblies that are added to the Plugins.xml file that is located at the install location of WinTAK.

Copy assembly in post build event

In the project properties, view the Build Events tab and add the following to the post build event:

xcopy "$(TargetDir)$(TargetName).dll" "C:\Program Files(x86)\WinTak\Plugins\" /y
xcopy "$(TargetDir)$(TargetName).pdb" "C:\Program Files(x86)\WinTak\Plugins\" /y

Make sure you are running Visual Studio with Administrator privileges or this post build step may fail to run.

Updating the Plugins.xml file

Open the Plugins.xml file and add the following with the Plugins element

<Plugin path='Plugins/SamplePlugin.dll' />

This can also be done by running WinTAK and navigating to Settings->Tool Preferences->Plugin Management and selecting the checkbox next to your plugin.

Debugging your plugin

In the project properties view the Debug tab. Set the Start Action to be Start external program and set the location to be WinTAK.exe Under Start Options set the Working directory to be the install location of WinTAK.

Debugging your Plugin

From here you can debug and step into the code of your custom Button and DockPane.

Adding map functionality to your DockPane

Updating the view

We will add a ToggleButton and one TextBlock. The ToggleButton will (un)subscribe to map mouse movements and the TextBlock will display the map coordinates.

Add the following XAML to the view inside of the Grid element.

<StackPanel>
<ToggleButton Content="(de)activate" IsChecked="{Binding IsActive}"/>
<TextBlock>
<Run Text="{Binding Lat}"/>
<Run Text=","/>
<Run Text="{Binding Lon}"/>
</TextBlock>
</StackPanel>

Updating the DockPane

When the ToggleButton is clicked, the DockPane will subscribe or unsubscribe from map mouse movements. When the mouse is moved over the map, the DockPane will update the Lat and Lon properties with the current location.

Update the SampleDockPane class with the following:

[DockPane(ID, "Sample", Content = typeof(SampleView))]
class SampleDockPane : DockPane
{
internal const string ID = "Sample_SampleDockPane";
private double _lon;
private double _lat;
private bool _isActive;
public SampleDockPane()
{
}
public bool IsActive
{
get { return _isActive; }
set
{
SetProperty<bool>(ref _isActive, value, "IsActive");
if (value)
{
MapViewControl.MapMouseMove += MapViewControl_MapMouseMove;
}
else
{
MapViewControl.MapMouseMove -= MapViewControl_MapMouseMove;
}
}
}
public double Lon
{
get { return _lon; }
set { SetProperty(ref _lon, value); }
}
public double Lat
{
get { return _lat; }
set { SetProperty(ref _lat, value); }
}
private void MapViewControl_MapMouseMove(object sender, MapMouseEventArgs e)
{
Lat = e.WorldLocation.latitude;
Lon = e.WorldLocation.longitude;
}
}

Installing your Plugin

Third party plugins should be added to the WinTAK installer as optional plugins/components. Once your plugin has reached a Beta phase and is ready for testing notify the WinTAK development team that you would like it to be added to the WinTAK installer.
